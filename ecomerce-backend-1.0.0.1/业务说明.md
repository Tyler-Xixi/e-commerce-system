# 电商供应链一体化平台说明文档

## 项目概述

本项目是一个前后端分离的电商供应链管理系统，核心目标是围绕企业采购寻源流程，把品类工程师、品类组长、采购处长、供应商操作员、仓储调度员 5 类角色连接到同一套业务闭环中。

系统主流程：

1. 品类工程师创建寻源需求。
2. 品类组长审核需求。
3. 审核通过后发布 RFQ 询价单。
4. 供应商查看 RFQ 并提交报价。
5. 品类工程师在报价汇总中按价格排序比价，生成商务定点单。
6. 采购处长审批商务定点。
7. 审批通过后触发采购订单、付款结算、入库和物流调度流程。

## 常见问题

### 前端提示后端未连接

检查后端是否启动：

```text
http://localhost:8080/api/public/health
```

检查前端代理：

```text
frontend/vite.config.js
```

### 登录失败

优先检查：

1. 数据库是否导入完整 SQL。
2. `sys_user.password` 是否有 BCrypt 密码。
3. 后端 `application.yml` 的数据库账号密码是否正确。
4. 后端控制台是否有 SQL 或认证错误。

### 页面没有数据

检查：

1. 是否导入 `e_commerce_system_complete.sql`。
2. 后端是否连接到正确数据库。
3. 当前角色是否有对应模块权限。
