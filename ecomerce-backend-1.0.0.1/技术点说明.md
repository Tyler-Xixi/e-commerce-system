# 后端技术检查报告

检查项目：`ecomerce-backend-1.0.0.1`  
检查时间：2026-05-31  
结论：项目已经接入 Redis、JWT、用户密码加密、分页查询、Spring Security 细粒度权限配置。

## 一、当前后端技术栈

项目是 Java 17 + Spring Boot 3.2.0 后端，主要使用：

- Spring Web：提供 REST API。
- Spring Data JPA：数据库 ORM 和分页查询。
- Spring Security：认证、鉴权、方法级权限控制。
- JWT：无状态登录令牌。
- Redis：登录 token 状态管理、权限缓存。
- MySQL：业务数据库。
- Lombok：减少实体、服务类样板代码。
- Jackson：JSON 序列化、日期格式处理。
- Maven：项目构建管理。

主要依赖位置：`pom.xml`

## 二、Redis 使用情况

结论：已使用。

相关文件：

- `pom.xml`
  - 引入 `spring-boot-starter-data-redis`。
- `src/main/resources/application.yml`
  - 配置 `spring.data.redis.host`、`port`、`database`、`timeout`、Lettuce 连接池。
- `src/main/java/com/ecommerce/config/RedisConfig.java`
  - 配置 `RedisTemplate<String, Object>`。
  - Key 使用 `StringRedisSerializer`。
  - Value 和 HashValue 使用 `GenericJackson2JsonRedisSerializer`。
- `src/main/java/com/ecommerce/service/AuthTokenService.java`
  - 使用 `StringRedisTemplate` 保存、校验、删除登录 token。
  - Redis key 前缀：`ecommerce:auth:token:`。
  - TTL 与 JWT 过期时间保持一致。
- `src/main/java/com/ecommerce/service/MenuService.java`
  - 使用 Redis 缓存角色权限。
  - Redis key 前缀：`ecommerce:permission:`。
  - 权限缓存时间：30 分钟。

实现说明：

1. 用户登录成功后，后端生成 JWT，并取出 JWT 的 `jti` 作为 tokenId。
2. `AuthTokenService.storeToken(...)` 将 tokenId、userId、username、roleCode、expiresAt 写入 Redis，并设置过期时间。
3. 每次请求经过 `JwtAuthenticationFilter` 时，先解析 JWT，再调用 `AuthTokenService.isTokenActive(tokenId)` 检查 token 是否仍在 Redis 中。
4. 用户退出登录时，`AuthService.logout(...)` 调用 `AuthTokenService.revokeToken(...)` 删除 Redis 中的 token，实现服务端主动失效。
5. 角色权限通过 `MenuService.getPermissions(roleCode)` 获取，并缓存在 Redis 中，减少频繁查询菜单权限表。

## 三、JWT 使用情况

结论：已使用。

相关文件：

- `pom.xml`
  - 引入 `jjwt-api`、`jjwt-impl`、`jjwt-jackson`。
- `src/main/resources/application.yml`
  - 配置 `jwt.secret`、`jwt.expiration`、`jwt.header`。
- `src/main/java/com/ecommerce/util/JwtUtil.java`
  - 负责 JWT 生成、解析、获取用户信息、获取过期时间、获取 tokenId。
- `src/main/java/com/ecommerce/security/filter/JwtAuthenticationFilter.java`
  - 负责从请求头读取 `Authorization: Bearer <token>`。
  - 校验 JWT 和 Redis token 状态。
  - 构建 Spring Security 认证对象。
- `src/main/java/com/ecommerce/service/AuthService.java`
  - 登录成功后调用 `JwtUtil.generateToken(...)` 生成 token。
- `src/main/java/com/ecommerce/controller/AuthController.java`
  - 提供 `/auth/login`、`/auth/logout`、`/auth/me` 接口。

实现说明：

1. 登录接口校验用户名和密码。
2. 认证成功后，`JwtUtil.generateToken(...)` 生成 JWT。
3. JWT 中包含：
   - `userId`
   - `username`
   - `roleCode`
   - `jti`
   - `iat`
   - `exp`
4. 请求进入系统时，`JwtAuthenticationFilter` 解析 token，读取用户角色和权限。
5. 过滤器创建 `JwtPrincipal`，再构造 `UsernamePasswordAuthenticationToken` 放入 `SecurityContextHolder`。
6. 后续接口上的 `@PreAuthorize` 根据当前用户权限决定是否放行。

## 四、数据库用户密码加密情况

结论：已使用 BCrypt 加密用户登录密码。

相关文件：

- `src/main/java/com/ecommerce/security/SecurityConfig.java`
  - 注册 `PasswordEncoder` Bean：`BCryptPasswordEncoder`。
- `src/main/java/com/ecommerce/service/AuthService.java`
  - 登录时使用 `passwordEncoder.matches(rawPassword, encodedPassword)` 校验密码。
- `src/main/java/com/ecommerce/config/ApplicationInitializer.java`
  - 初始化默认用户时使用 `passwordEncoder.encode("123456")` 加密密码后入库。
- `src/main/resources/db/update_users.sql`
  - 示例用户密码字段已经是 BCrypt hash。
- `src/main/java/com/ecommerce/entity/SysUser.java`
  - `password` 字段长度为 200，能够保存 BCrypt hash。

实现说明：

1. 系统不会直接比较明文密码。
2. 初始化用户时，默认密码 `123456` 会经过 BCrypt 加密后写入 `sys_user.password`。
3. 登录时，用户提交的明文密码会通过 `PasswordEncoder.matches(...)` 与数据库中的 BCrypt hash 比对。
4. BCrypt 自带随机盐，同一个明文密码每次加密后的结果通常不同，比普通 MD5/SHA 摘要更适合保存登录密码。

注意事项：

- 当前用户登录密码已加密。
- 但 `src/main/resources/application.yml` 中的数据库连接密码 `spring.datasource.password` 仍是明文 `123456`。如果你说的“数据库密码加密”是指数据库连接密码加密，那么当前还没有做，建议改成环境变量、配置中心密文或 Jasypt。

## 五、分页查询使用情况

结论：已使用。

相关文件：

- `src/main/java/com/ecommerce/util/PageRequestUtil.java`
  - 统一处理分页参数。
  - 默认页码：1。
  - 默认每页数量：10。
  - 最大每页数量：1000。
  - 默认按 `id` 倒序。
- `src/main/java/com/ecommerce/dto/PageResult.java`
  - 统一分页响应结构。
  - 兼容 `records/content`、`total/totalElements`、`pages/totalPages` 字段。
- `src/main/java/com/ecommerce/controller/PlatformController.java`
  - `/users`
  - `/suppliers`
  - `/products`
  - `/orders`
  - `/inventory`
  - `/logistics`
  - `/payments`
  - `/aftersales`
- `src/main/java/com/ecommerce/controller/SourcingController.java`
  - `/sourcing/demands`
  - `/sourcing/quotations`
  - `/sourcing/quotes`
  - `/sourcing/awards`
  - `/sourcing/award-notices`
  - `/sourcing/tasks/pending/{userId}`
  - `/sourcing/notifications/{userId}`
- 多个 Repository 使用 Spring Data JPA `Page`、`Pageable` 查询。

实现说明：

1. 控制器接收 `page`、`size` 请求参数。
2. `PageRequestUtil.of(page, size)` 将参数转换成 JPA 的 `Pageable`。
3. Repository 执行分页查询并返回 `Page<T>`。
4. `PageResult.of(page)` 将 JPA 分页对象转换为前端统一分页格式。
5. 前端可以通过 `records/content` 获取数据，通过 `total/totalElements`、`page`、`size`、`pages/totalPages` 获取分页信息。

## 六、Spring Security 细粒度权限配置

结论：已使用，并且已经做到接口级权限控制。

相关文件：

- `src/main/java/com/ecommerce/security/SecurityConfig.java`
  - 启用 `@EnableWebSecurity`。
  - 启用 `@EnableMethodSecurity`。
  - 配置无状态会话：`SessionCreationPolicy.STATELESS`。
  - 关闭表单登录、HTTP Basic、CSRF。
  - 放行 `/auth/login`、`/public/**`、`OPTIONS`。
  - 其他接口必须认证。
  - 将 `JwtAuthenticationFilter` 加入 Spring Security 过滤器链。
- `src/main/java/com/ecommerce/security/filter/JwtAuthenticationFilter.java`
  - 解析 JWT。
  - 校验 Redis token 状态。
  - 加载角色权限。
  - 写入 Spring Security 上下文。
- `src/main/java/com/ecommerce/security/JwtPrincipal.java`
  - 将 `roleCode` 转成 `ROLE_JSxx`。
  - 将权限字符串转成 `GrantedAuthority`。
- `src/main/java/com/ecommerce/security/RolePermissionRegistry.java`
  - 内置角色默认权限。
  - 角色包括 `JS01`、`JS02`、`JS03`、`JS04`、`JS05`、`JS06`。
- `src/main/java/com/ecommerce/service/MenuService.java`
  - 从 `sys_role_menu`、`sys_menu` 读取角色菜单权限。
  - 与内置默认权限合并。
  - 使用 Redis 缓存。
- `src/main/resources/db/init_menu.sql`
  - 创建 `sys_menu`、`sys_role_menu`。
  - 初始化菜单、权限标识、角色菜单关系。
- `src/main/java/com/ecommerce/controller/PlatformController.java`
  - 使用大量 `@PreAuthorize("hasAuthority('...')")` 控制平台模块接口。
- `src/main/java/com/ecommerce/controller/SourcingController.java`
  - 使用 `@PreAuthorize("hasAuthority('...')")` 控制寻源流程接口。

实现说明：

1. 用户登录后拿到 JWT。
2. 请求携带 JWT 进入后端。
3. `JwtAuthenticationFilter` 解析出 `roleCode`。
4. `MenuService.getPermissions(roleCode)` 获取该角色拥有的权限集合。
5. `JwtPrincipal` 将权限集合转成 Spring Security 的 `GrantedAuthority`。
6. 控制器方法上的 `@PreAuthorize` 判断当前用户是否拥有对应权限。
7. 没有登录返回 401；没有权限返回 403。

示例：

- 查看供应商列表需要 `supplier:list`。
- 新增、编辑、删除供应商需要 `supplier:manage`。
- 查看库存需要 `warehouse:inventory`。
- 管理库存需要 `warehouse:inventory:manage`。
- 审批需要 `dashboard:approval`。
